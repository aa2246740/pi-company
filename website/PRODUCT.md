# Product

## Register

product

## Users

**主要用户**: 开发者和技术团队负责人
- 对 AI 代理协作感兴趣的技术人员
- 希望了解和使用 pi-company 的开发者
- 需要本地多代理协作方案的团队
- 喜欢终端风格界面的技术爱好者

**用户上下文**:
- 在本地开发环境中工作
- 使用 macOS/Linux/Windows 终端
- 熟悉命令行工具和开发工作流
- 寻求高效、可控的 AI 代理协作方案

**用户目标**:
- 理解 pi-company 的核心概念和价值
- 学习如何安装、配置和使用 pi-company
- 掌握完整的代理协作工作流
- 通过交互式教程获得实践经验

## Product Purpose

pi-company 网站是 pi-company 的官方文档和产品体验平台。

**核心目标**:
1. **教育**: 教会用户如何使用 pi-company，而不仅仅是营销
2. **展示**: 通过交互式教程展示完整的代理协作工作流
3. **指导**: 提供从零到合并的完整学习路径
4. **验证**: 作为 pi-company 能力的真实验证项目

**成功标准**:
- 用户能够从零开始学习 pi-company 的完整工作流
- 交互式教程真实可用，而非装饰性占位符
- 所有主要功能都有详细的教程或指南
- 网站能够本地运行并通过生产构建

## Brand Personality

**三个关键词**: 专业、复古未来、可控

**语调特点**:
- **专业**: 技术准确、信息密集、不废话
- **复古未来**: 终端美学、CRT 荧光色、像素感
- **可控**: 强调用户对代理的可见性和可操作性

**情感目标**:
- 信任感: 展示 pi-company 的可靠性和透明度
- 掌控感: 用户能够完全控制代理行为
- 效率感: 展示代理协作带来的效率提升
- 怀旧感: 复古终端风格带来的技术怀旧

**声音特征**:
- 不使用营销术语（如"赋能"、"超级充电"、"无缝"）
- 不使用夸张的承诺
- 使用具体的技术术语和命令
- 保持简洁、直接、技术准确

## Anti-references

**明确避免的设计**:
1. **通用 SaaS 着陆页**: 避免"英雄区域 + 特性网格 + 定价表"的模板
2. **单一绿色调色板**: 不要只用绿色，要使用完整的 CRT 荧光色系
3. **装饰性元素**: 避免装饰性斑点/球体、通用渐变、玻璃态效果
4. **营销废话**: 避免"赋能"、"超级充电"、"无缝体验"等空洞词汇
5. **扁平化设计**: 不要过度简化，保持终端风格的密度感
6. **过度动画**: 避免无意义的动画，保持专业感

**参考网站特征** (避免的):
- Stripe/Linear/Vercel 的营销着陆页风格
- 纯文档站点（如 VuePress 默认主题）
- 传统企业软件界面
- 现代 SaaS 仪表板

## Design Principles

1. **实践所宣讲**: 网站本身应该展示 pi-company 的能力，使用 pi-company 的真实概念和术语

2. **展示而非讲述**: 通过交互式模拟器和教程展示功能，而不仅仅是文字描述

3. **专家信心**: 界面应该传达技术专业性，让用户相信这是一个成熟、可靠的工具

4. **信息密度优先**: 终端风格的核心是信息密度，每个像素都应该传递有价值的信息

5. **真实透明**: 不隐藏复杂性，展示真实的工作流程，包括错误和边界情况

6. **渐进式披露**: 从简单概念开始，逐步引入高级功能，但不要过度简化

## Accessibility & Inclusion

**WCAG 级别**: AA 级标准

**已知用户需求**:
- 键盘导航: 所有交互组件必须支持键盘操作
- 屏幕阅读器: 使用语义化 HTML 和 ARIA 属性
- 颜色对比: 确保文本对比度符合 WCAG AA 标准（4.5:1 对比度）
- 动画控制: 提供减少动画的选项（`prefers-reduced-motion`）
- 字体大小: 支持用户自定义字体大小
- 焦点指示: 清晰的焦点状态指示

**特殊考虑**:
- CRT 效果可能对某些用户造成不适，提供关闭选项
- 高对比度模式增强可读性
- 确保在明亮环境下也能正常阅读
- 支持暗色和高对比度主题

**国际化**:
- 当前主要支持英文（pi-company 是英文产品）
- 预留国际化扩展能力
- 使用清晰、简单的英文，避免俚语和复杂句式
